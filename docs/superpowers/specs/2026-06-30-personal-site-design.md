> 日期：2026-06-30
> 状态：设计已确认，等待实现

## 1. 项目概述

在 `807544076.github.io` 上搭建一个**日系简约风格**的个人站点，包含首屏动态背景、博客、项目展示、隐藏彩蛋，并预留 Live2D 看板娘和小游戏接口。

### 核心原则

- 纯前端，完全静态，无构建工具
- 日系简约视觉风格（米白底色 + 樱花粉强调）
- 功能模块化，每个独立功能一个 JS 文件
- 渐进增强：首屏优先加载，次要功能延迟加载
- 彩蛋不干扰正常浏览体验

## 2. 站点结构

```
/
├── index.html                     # 首页（首屏 + 内容区）
├── css/
│   ├── style.css                  # 全局样式、配色、排版
│   ├── hero.css                   # 首屏专用样式（毛玻璃卡片、粒子层）
│   └── masonry.css                # 瀑布流/网格样式
├── js/
│   ├── main.js                    # 入口初始化，协调各模块加载
│   ├── particles.js               # Canvas 粒子背景引擎
│   ├── easter-egg.js              # 彩蛋系统（控制台 + 页面交互）
│   ├── masonry.js                 # 瀑布流布局渲染
│   ├── live2d-bridge.js           # Live2D 预留接口（后期实现）
│   └── games.js                   # 小游戏预留入口框架（后期实现）
├── blog/
│   ├── index.html                 # 博客列表页（可选，与首页内容区保留一种方式）
│   └── posts.json                 # 博文元数据索引
├── projects/
│   ├── index.html                 # 项目列表页（可选）
│   └── projects.json              # 项目元数据索引
└── assets/
    ├── images/                    # 图片资源
    └── fonts/                     # 字体文件（如有自定义字体）
```

### 页面路由

| 路径 | 内容 | 说明 |
|------|------|------|
| `/` | 首页 | 首屏 + 博客区 + 项目区 + 页脚 |
| `/blog/post-name.html` | 博文详情 | 独立 HTML 页面 |
| `/projects/project-name.html` | 项目详情 | 独立 HTML 页面 |
| `/games/` | 游戏入口页 | 占位，后期实现 |

所有页面通过直接文件路径访问，不使用 JS 路由。新增页面只需创建新的 HTML 文件。

## 3. 首页布局

### 3.1 首屏（Hero Section）

```
┌──────────────────────────────────────┐
│        樱花粒子动态背景 (Canvas)       │
│                                      │
│        ┌──────────────────┐          │
│        │   头像/Logo 占位   │          │
│        │   Oceanfish       │ ← 毛玻璃 │
│        │   一句话简介       │   卡片   │
│        │                  │          │
│        │    ▼ Scroll      │          │
│        └──────────────────┘          │
└──────────────────────────────────────┘
```

**樱花粒子背景**（`particles.js`）：
- Canvas 全屏绘制樱花花瓣粒子，缓缓飘落带旋转和摇摆
- 鼠标划过时花瓣轻微散开避开指针
- 鼠标点击时产生波纹/花瓣炸散效果
- 帧率：桌面 60fps / 移动端 30fps
- 页面不可见时暂停渲染

**毛玻璃个人卡片**：
- `background: rgba(255, 255, 255, 0.6)` + `backdrop-filter: blur(20px)`
- 居中显示，半透明背景透出粒子动态
- 头像占位、昵称、一句话简介
- 鼠标 hover 时卡片上浮 4px，阴影加深

**滚轮引导**：
- 卡片下方跳动箭头 "Scroll ▼"
- 点击平滑滚到内容区

### 3.2 内容区（Content Section）

首屏下滑后，内容区分为两个独立区块，中间以分隔线隔开：

**博客区 — 瀑布流布局**：
- 卡片高度参差，呈现日系杂志式排列
- 每张卡片：标题 + 摘要 + 日期 + "📝 Blog" 标签
- 点击卡片跳转到 `/blog/xxx.html`
- 数据来源：`blog/posts.json`

**项目区 — 网格布局**：
- 规整网格（2-3 列），项目数量通常较少
- 每张卡片：项目名称 + 简述 + GitHub 链接
- 点击卡片跳转到 `/projects/xxx.html`
- 数据来源：`projects/projects.json`

### 3.3 页脚

- 版权信息：`❮ Oceanfish © 2026 ❯`
- 社交媒体链接（如 GitHub）
- "🎮 Games" 入口文字链接

## 5. 彩蛋系统

### 5.1 控制台彩蛋

- **触发时机**：用户打开 DevTools → Console 标签页
- **内容**：自动输出 Oceanfish 定制的彩色 ASCII Art 标题
- **结尾**：附带一行线索文字（暗示页面交互彩蛋的触发方式）

### 5.2 页面交互彩蛋 A — 樱花绽放

- **触发条件**：5 秒内连续点击页面空白处 5 次
- **效果**：粒子数量暴增 3 倍，持续 10 秒后缓出恢复
- **可重复触发**

### 5.3 页面交互彩蛋 B — 午夜模式

- **触发条件**：浏览器时间在凌晨 0:00~1:00 之间加载页面
- **效果**：
  - 粒子颜色切换为银白月光色系
  - 背景替换为夜樱图片
  - 卡片色调微冷、阴影加深
- **用户感知**：控制台彩蛋中可能提示"也许深夜再来看看？"

### 5.4 页面交互彩蛋 C — 隐藏留言

- **触发条件**：在控制台输入 `oceanfish.secret()`
- **效果**：页面上浮显示一段预设的隐藏文字
- **样式**：日系风格的浮层，5 秒后自动淡出

## 7. 性能与加载策略

| 优先级 | 资源 | 策略 |
|--------|------|------|
| 首屏关键 | 首屏 CSS（内联到 `<head>`） | 同步，不阻塞渲染 |
| 首屏关键 | 个人卡片 HTML | 直接在页面中静态编写 |
| 首屏关键 | 纯色后备背景 | Canvas 未启动前显示 |
| 延迟加载 | `particles.js` | 页面 load 后 200ms 启动 |
| 按需加载 | `masonry.js` + 内容数据 | IntersectionObserver 检测到滚动到内容区时加载 |
| 延迟加载 | `easter-egg.js` | 页面完全加载后 3 秒加载 |
| 懒加载 | 图片 | `loading="lazy"` |
| 暂停 | Canvas 渲染 | 页面不可见时停止（Page Visibility API） |

### 移动端优化

- 粒子数量减少 60%
- 瀑布流降级为单列布局
- backdrop-filter 根据支持情况降级
- 触摸事件替代鼠标 hover

## 8. 全局交互规范

| 元素行为 | 效果 | 实现 |
|---------|------|------|
| 卡片 hover | 上浮 4px + 阴影加深 | CSS transition 0.3s |
| 链接 hover | 下划线从中间展开 | CSS `background-size` 动画 |
| 点击反馈 | 微缩放 0.97 → 恢复 | CSS transform |
| 页面滚动 | 平滑滚动 | `scroll-behavior: smooth` |
| 滚轮提示 | 闪烁动画，点击隐藏 | CSS animation |
| 内容切换 | 淡入淡出 | CSS opacity transition |
